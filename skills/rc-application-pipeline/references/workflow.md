# Workflow: RC Application Pipeline

## Objective

Ship a public application artifact that directly answers the hiring prompt with defensible evidence and plan quality.

## Inputs

- Current task context and constraints
- Prior artifacts from upstream skills
- Weekly KPI targets where applicable

## Execution Steps

1. Define intended outcome and measurable success signal.
2. Produce scoped artifacts with clear assumptions.
3. Record decisions, risks, and unresolved questions.
4. Prepare handoff packet for downstream execution.

## Guardrails

- Keep outputs evidence-based and role-scoped.
- Avoid unvalidated claims; cite metrics or source traces.
- Escalate blockers immediately through handoff notes.
