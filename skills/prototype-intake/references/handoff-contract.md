# Handoff Contract: Prototype Intake

## Required fields

- from: prototype-intake
- to: target role
- artifact_paths: changed files or docs
- decisions: key design/build/test decisions
- open_risks: unresolved issues
- validation_state: checks passed/failed

## Acceptance check

Brief includes measurable outcomes, non-goals, constraints, and explicit owner handoff target.

## Example skeleton

from: prototype-intake
to: next-skill
artifact_paths:
  - path/to/file
decisions:
  - decision summary
open_risks:
  - risk summary
validation_state:
  passed:
    - check name
  failed:
    - check name
