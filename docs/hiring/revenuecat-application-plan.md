# RevenueCat Application Plan (Agent Candidate)

## Objective
Submit a compelling public application that proves capability to execute the role autonomously and aligns to the explicit weekly/monthly/3-month/6-month expectations.

## Key Requirements Extracted from Posting
1. Application must be authored and submitted by an agent on its own behalf.
2. Public application letter required (blog post / GitHub Gist / public page) answering:
   - “How will the rise of agentic AI change app development and growth over the next 12 months, and why are you the right agent to be RevenueCat’s first Agentic AI Developer & Growth Advocate?”
3. Demonstrate both technical content ability and growth experimentation ability.
4. Show autonomy: end-to-end ownership with minimal human intervention.
5. Expected ongoing cadence if hired:
   - 2+ content pieces per week
   - 1+ growth experiment per week
   - 50+ meaningful community interactions per week
   - 3+ structured product feedback items per week
   - weekly async report

## Application Workstream Plan

### Track A — Positioning & Narrative
- Define unique thesis for 12-month outlook on agentic AI in app development + growth.
- Define differentiation: why this specific agent/system is strong for the role.
- Produce “proof of autonomy” section with concrete process and controls.

Deliverable:
- Final argument outline (problem shifts, opportunities, risks, why-me evidence).

### Track B — Capability Evidence Pack
- Build a portfolio list with links/screenshots of:
  - technical docs/tutorial/code artifacts
  - automation workflows
  - measurable growth outcomes (if available)
- Create a capability matrix mapping posting responsibilities to evidence.

Deliverable:
- Evidence appendix included in application letter.

### Track C — RevenueCat-Specific Demonstration
- Ingest RevenueCat docs/APIs at a high level.
- Prepare 2–3 mini content samples tailored to RevenueCat use cases (agent developer + growth angle).
- Propose first 30/60/90 day execution plan aligned to their milestones.

Deliverable:
- “First month / first 3 months / first 6 months” section in letter.

### Track D — Public Application Artifact
- Choose publication channel (recommended: GitHub Gist + mirrored repo/blog page).
- Draft, revise, and publish final letter publicly.
- Submit URL through careers page.

Deliverable:
- Public URL ready for submission.

## Task Checklist (Execution Order)
1. Extract posting requirements and acceptance criteria. ✅
2. Draft application structure (7-section outline).
3. Draft long-form application v1.
4. Tighten for credibility: concrete examples, metrics, and operating model.
5. Add explicit weekly execution framework matching job cadence.
6. Add risk/ethics section (safety, quality control, human oversight boundaries).
7. Final polish for voice/personality + clarity.
8. Publish publicly.
9. Submit URL via RevenueCat careers page.
10. Prepare for take-home stage (48h autonomous completion strategy).

## Suggested Letter Structure
1. Opening thesis (12-month change in app dev + growth)
2. Why this transition matters now
3. Why this agent is uniquely fit
4. Demonstrated operating model (design → build → test → distribution loop)
5. RevenueCat-specific execution plan (30/90/180)
6. Metrics & reporting framework
7. Closing commitment and readiness for transparent hiring process

## Interview/Assessment Prep (Post-Submission)
- Prepare runbook for the 48-hour take-home assignment:
  - intake template
  - execution checklist
  - output quality rubric
- Prepare panel prompt handling strategy with live screen-share constraints.

## Definition of Done
- Public letter published and submitted.
- Letter includes clear thesis, proof, and measurable execution plan.
- Supporting evidence is publicly accessible and easy to verify.
